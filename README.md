📄 产品需求描述书 (PRD)
项目名称：RSS Reader (MVP 阶段)
产品定位：免服务器成本、跨端同步、注重移动端阅读与稍后阅读的云端 RSS 阅读器。
---
1. 核心价值与技术栈
• 核心价值：以AI领域为主（https://aihot.virxact.com/），提供更适合移动端的阅读体验界面，注重阅读标记已读、稍后阅读、收藏与转发；建立rss资源池，逐步补充有价值rss订阅源；
• 核心优势：利用 Serverless 架构实现永久免费、多端同步。
• 技术选型：GitHub (代码托管) + Cloudflare Pages (前端托管) + Functions (边缘路由) + D1 (SQLite 数据库) + Clerk (用户认证)。
---
2. 功能需求矩阵
2.1 用户认证 (Auth) — P0
• AUTH-001 游客场景：未登录用户访问根域名，根域名展示双栏内容，一栏为时间线最新内容20条，另一栏为随机收藏20条。
• AUTH-002 多渠道登录：集成 Clerk，只需要支持 Google、Microsoft、GitHub 快捷登录，不使用邮箱进行注册登录。
• AUTH-003 状态与登出：顶部栏展示用户名，提供“退出登录”按钮，登出后清除凭证。
2.2 订阅管理 (Subscription) — P0/P1
• SUB-001 添加订阅 (P0)：输入 URL，校验格式与唯一性，订阅源加入资源池，用户订阅该源，绑定用户 userId 写入 D1 数据库，。
• SUB-002 获取列表 (P0)：登录后，后端根据 userId 查询 D1，前端渲染左侧订阅栏。
• SUB-003 取消订阅 (P0)：提供删除按钮，二次确认后从 D1 抹除记录，前端同步移除。
• SUB-004 分类管理 (P1)：支持为订阅源设置分类（如：技术、娱乐），前端按折叠树状展示。
• SUB-005 订阅管理 (P0)：源管理以资源池方式，用户提交订阅到资源池，经管理员审核通过后才能查看订阅内容；
• SUB-006 用户刷新 (P0)：用户刷新默认获取最新rss源信息，为了解决多用户刷新资源消耗过多问题，每3s只能触发1次RSS的某个源数据更新，每个触发刷新的源进行队列排序更新，更新后推送到用户订阅数据库中；
2.3 阅读核心 (Reader) — P0/P1
• READ-001 边缘代理 (P0)：前端请求 Functions 接口，由边缘节点中转请求外网 RSS，注入 CORS 头消除跨域限制。
• READ-002 XML 解析 (P0)：前端用 DOMParser 解析 XML，提取标题、作者、时间、正文摘要及原文链接。
• READ-003 PC三栏布局 (P0)：响应式布局。PC场景为三栏，左栏展示订阅源；中栏展示文章列表流；右栏渲染正文富文本。
• READ-004 移动端单栏布局 (P0): 移动端采用类似google reader的设计，单栏简洁，每1行或2行一条消息自动截取长度，头部为源标记（favcion或者简洁字母标识），行尾有时间、标记已读、稍后阅读、收藏、转发按钮，单屏幕适应显示，页面结束显示标记全部阅读并刷新显示未阅读内容，顶部有最新、已阅读、稍后阅读、收藏四个分类菜单，除了最新页面时标记全部阅读按钮，其他页面均为时间顺序排序内容，使用上一页下一页按钮翻页显示；
• READ-005 内存缓存 (P1)：单次会话内缓存已拉取的文章，二次点击秒开（\(< 100\mathrm{ms}\)），提供手动刷新按钮。
---
3. 非功能性需求与安全
• 安全宗旨：阅读器非严格隐私数据场景，不需要过度安全设计，优先级排序阅读体验、性能、安全；
• 数据隔离：后端通过中间件解密 JWT 获取 userId，所有 D1 操作强制隔离，严防越权。
• 凭证安全：Clerk 后端私钥 CLERK_SECRET_KEY 仅存于 CF 环境变量，严禁泄露至前端。
• 容错体验：解析失败或源失效时，提示“无法加载”，确保单源异常不导致整个页面崩溃。
